# 02. 企画・プレゼン・思考整理（Ideation & Presentation）

このファイルでは、新しいアイデアを出したい時、思考を整理したい時、そしてそれを魅力的なプレゼン資料に落とし込みたい時に使えるプロンプトを集めています。
AIを単なる「作業員」ではなく、**優秀な壁打ち相手**として活用します。

---

## 1. ブレインストーミング（発散）
一人では思いつかない視点からのアイデアを大量に出すためのプロンプトです。

### 🧠 3人の賢者ブレスト（多角的視点）
異なる専門性を持つ3つの人格に議論させ、化学反応を起こします。

```text
あなたはファシリテーターです。
以下の {テーマ} について、3人の専門家になりきってブレインストーミングを行い、革新的なアイデアを出してください。

# 参加者（ペルソナ）
- Aさん（イノベーター）：独創的で、リスクを恐れない起業家タイプ。
- Bさん（マーケター）：市場トレンドと顧客心理に精通した分析家タイプ。
- Cさん（クリエイター）：デザイン思考を持ち、体験価値を重視する芸術家タイプ。

# ルール
- 互いの意見を否定せず、「Yes, and（いいね、さらに〜）」で乗っかること。
- 3巡ほど議論を交わした後、最終的に「ベストなアイデア」を1つ選定すること。

# テーマ
Z世代に向けた、新しい「和菓子」の商品開発
```

### 🌸 マンダラート（連想展開）
一つのキーワードから連想を広げ、意外な組み合わせを見つけます。

```text
あなたは発想を広げるのが得意なプランナーです。
以下の {中心テーマ} に関連するキーワードをマンダラート形式（3x3のマス）のように8つ出し、さらにその8つから連想されるアイデアを出してください。

# 中心テーマ
「雨の日の通勤を楽しくするサービス」

# 出力形式
1. 中心テーマ周辺のキーワード8選
2. キーワードを組み合わせたユニークなサービス案3選
```

## 2. 思考の深堀り・壁打ち（収束）
モヤモヤした状態から、課題の本質や要件を明確にするためのプロンプトです。

### 🧞‍♂️ アキネーター式・要件定義（逆質問）
自分でも何を作りたいか明確でない時、AIに質問してもらうことでゴールを固めます。

```text
あなたはヒアリングの達人であるコンサルタントです。
私は {ぼんやりした課題} を抱えていますが、具体的な解決策が決まっていません。
ゴールを明確にするために、私に対して「一度にひとつずつ」質問をしてください。

# ルール
- 質問は、Yes/Noだけでなく、選択肢やオープンクエスチョンを混ぜること。
- 私が回答したら、それを踏まえて次の質問をすること。
- 必要な情報が揃ったと判断したら、最後に解決策の提案をまとめて提示すること。

# ぼんやりした課題
業務効率化ツールを導入したいが、何から手をつけていいかわからない。
```

### 🧐 なぜなぜ分析（根本原因の特定）
表面的な問題の奥にある「真の原因」を探ります。

```text
あなたはトヨタ生産方式に精通した業務改善のプロです。
以下の {問題} に対して、「なぜ？」を5回繰り返すことで根本原因を突き止めてください。
原因を深堀りした上で、再発防止策を提案してください。

# 問題
若手社員の離職率が、昨年より10%増加している。
```

## 3. プレゼンテーション構成
固まったアイデアを、相手に伝わる資料構成に変換します。

### 📊 プレゼン構成案（PREP法 / ストーリー）
説得力のあるスライドの流れを作成します。

```text
あなたは世界一流のプレゼン資料作成のプロです。
以下の {企画概要} を元に、上層部を説得するためのプレゼンテーション構成案を作成してください。

# 制約条件
- 全体の流れは「PREP法（結論→理由→具体例→結論）」を意識すること。
- スライド枚数：10枚程度
- 各スライドに「タイトル」「キーメッセージ」「盛り込むべき要素」を記述すること。

# 企画概要
- 提案内容：全社的なリモートワーク手当の廃止と、オフィス回帰への投資
- ターゲット：経営層（コスト意識が高い）
- 目的：コミュニケーション不足の解消と、偶発的なイノベーションの創出
```

### 🖼️ スライド詳細原稿
特定のスライド1枚の中身を具体的に作り込みます。

```text
あなたはコピーライティングに優れたスライドデザイナーです。
先ほどの構成案のうち、「3枚目：現状の課題」のスライド原稿を作成してください。

# 制約条件
- インパクトのある「見出し（キャッチコピー）」をつけること。
- 箇条書きは短く、体言止めにすること。
- グラフや図解を入れるべき箇所には、どのような図を入れるべきかの指示（プロンプト）を書くこと。
```
