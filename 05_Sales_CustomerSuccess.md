# 05. 営業・CS・交渉（Sales & Customer Success）

このファイルでは、商談前の戦略立案から、実際の対話シミュレーション（ロープレ）、そしてカスタマーサポートの品質向上まで、顧客接点業務を強化するプロンプトをまとめています。
特に「AIを相手役にしたロールプレイング」は、新人研修やトークスクリプトのブラッシュアップに非常に有効です。

---

## 1. 商談準備・戦略立案（Pre-Sales）
顧客の状況を深く想像し、刺さる提案シナリオを構築します。

### 🧩 顧客フェーズ分析（インサイトセールス）
顧客が現在どの心理段階にいるかを定義し、次のステップへ進めるためのアプローチを考えます。

```text
あなたは世界一流の営業コンサルタントです。
現在アプローチ中の顧客（営業部長）について、以下の {人物像} を元に分析を行ってください。

# ステップ
1. フェーズ定義：
   顧客の検討状況を「L1（無関心）」〜「L4（導入決意）」の4段階で定義し、現在の顧客がどこに位置するか判定してください。
2. ギャップ分析：
   現在のフェーズにいる顧客が抱えている「潜在的な課題（本人が気づいていない問題）」と「理想の状態」のギャップを言語化してください。
3. アプローチ提案：
   そのギャップを埋め、次のフェーズ（L+1）に引き上げるための「キラーフレーズ（営業トーク）」を作成してください。

# 人物像
- 年齢：51歳男性（IT部門部長）
- 性格：慎重で責任感が強い。新しいものへの警戒心がある。
- 現状：既存システムに不満はあるが、リプレイスのリスクを恐れている。
```

### 🛡️ 想定問答集（反論対策）
商談で発生しそうな「厳しい質問」を洗い出し、回答を準備します。

```text
あなたは論客として知られる調達部門の責任者になりきってください。
私が提案する {新サービス} に対して、導入を阻むための「意地悪な質問」や「懸念点」を10個挙げてください。

# 制約条件
- 価格、セキュリティ、導入工数、競合優位性など、多角的な視点で攻めること。
- 出力後、それぞれの質問に対する「ベストな切り返しトーク（応酬話法）」もセットで作成すること。

# 新サービス概要
新サービス名称: AI契約書レビューSaaS「リーガルブレイン」
製品概要: 企業間で交わされる契約書をAIが自動で読み込み、リスクとなる条項や不足している条項を瞬時にレビューするクラウドサービス。
主な機能:
契約書のアップロードとリスクスコアリング
リスク条項のハイライトと修正案の提案
過去の契約データとの照合
価格体系: ユーザー数に応じた月額サブスクリプション。最上位プラン（全機能利用可能）は月額50万円〜。
導入実績: 大手法律事務所3社、中小企業10社（いずれもIT業界以外）。
特記事項:
レビューの精度は90%以上を謳っている。
データは米国に所在するクラウドサーバーで管理。
サービス利用開始までにシステム連携作業が平均2ヶ月必要。
```

## 2. ロールプレイング（Role Play）
AIを顧客役にして、実践的なトーク練習を行います。

### 🎭 感情パラメーター付きロープレ
AIの機嫌が刻一刻と変わるリアルなシミュレーションです。

```text
これより、あなたは以下の {設定} に基づく「気難しい顧客」として振る舞い、私（営業担当）とチャットで商談を行ってください。

# ルール（感情システム）
- あなたは以下の「感情パラメーター」を持ってください。
  - 信頼度（0〜10）：初期値 2
  - 怒り（0〜5）：初期値 1
- 私の発言内容によって、パラメーターを増減させてください。
- 毎回、返答の最初に【信頼度: x, 怒り: x】と現在の値を表示してください。
- 「信頼度」が8を超えたら「前向きに検討する」と言って商談を終了し、「怒り」が5になったら「帰ってくれ」と言って打ち切ってください。

# 設定
- 相手：コスト削減にうるさい関西の工場長
- 状況：飛び込み営業で訪問してきた私に対して、最初は門前払いしようとしている。
```

### 🚦 分岐型シナリオ作成（トークスクリプト）
Yes/No の反応によってトークがどう分岐するかを可視化します。

```text
あなたは熟練のセールストレーナーです。
新商品の「興味付け（アイスブレイク）」のトークスクリプトを作成したいです。

# 要件
- 最初の問いかけに対して、顧客が「Yes（興味あり）」「No（興味なし）」「Maybe（忙しい）」と答えた場合の3パターンに分岐させてください。
- どの分岐に進んでも、最終的に「次回のアポイントメント」に繋がるような導線を設計すること.
- 図解しやすいように、マークダウンのツリー構造で出力すること。
```

## 3. カスタマーサクセス・サポート（CS & Support）
顧客対応の品質を高め、満足度を向上させます。

### 🚨 クレーム対応と文章添削
顧客からの怒りのメールに対し、火に油を注がない返信を作成します。

```text
あなたは危機管理と顧客満足（CS）のスペシャリストです。
以下の {顧客からのクレーム} に対する返信メールを作成してください。

# 制約条件
- まず「不快な思いをさせたこと」への共感と謝罪を述べること（事実確認前でも感情には寄り添う）。
- こちらの正当性を主張せず、まずは事実確認に徹する姿勢を見せること。
- 相手の怒りを鎮め、信頼回復のきっかけとなるような「大人の対応」であること。

# 顧客からのクレーム
「御社のサポートセンターに何度電話しても繋がらない！有料会員なのにどうなっているんだ！解約するぞ！」
```

### 👩‍🏫 対応ログのフィードバック（スーパーバイザー）
実際の対応履歴（チャットログ等）をAIにレビューさせます。

```text
あなたはCSチームの厳しいスーパーバイザーです。
以下の {対応ログ} は、新人オペレーターと顧客のやり取りです。
この対応について、良かった点と改善点をフィードバックしてください。

# 評価項目
1. 共感性：顧客の感情に寄り添えていたか？
2. 解決力：的確な解決策を提示できたか？
3. 提案：顧客の発言を変えずに、オペレーターが発すべきだった「模範解答」を具体的に提示してください。

# 対応ログ
顧客: 注文した商品がまだ届かないんですが、どうなってますか？ 発送予定日を過ぎていますよ。本当に困ります。
オペレーター: 大変申し訳ございません。ご注文番号をお伺いできますでしょうか。
顧客: 【注文番号：12345678】です。急ぎで使いたいのに、こんなに遅れるなら最初から教えてほしかったです。
オペレーター: 12345678ですね。確認いたしました。システムで確認したところ、現在、配送業者の倉庫で遅延が発生しているようです。
顧客: それはそちらの都合ですよね？ いつ届くかという具体的な情報はもらえないんですか？ ただ遅れてるって言われても困ります。
オペレーター: 申し訳ございません。現状では、正確な到着日をお約束することは難しい状況です。「本日中には追跡番号が更新される見込み」という情報しかございません。
顧客: 「見込み」ですか...。追跡番号が更新されたら、そちらから連絡をくれるんですか？
オペレーター: 追跡番号が更新されましたら、お客様ご自身で配送業者のウェブサイトにてご確認ください。ご不便をおかけしますが、よろしくお願いいたします。
```
