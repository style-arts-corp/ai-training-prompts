# 01. 文書作成・業務効率化（Writing & Operations）

このファイルでは、全社員が日常的に行う**メール作成、文章校正、要約、会議準備**などの定型業務を、生成AIを使って劇的に効率化するためのプロンプトを集めています。
`00_Basic_Techniques.md` で学んだ「基本の型」を応用しています。

---

## 1. メール・チャット作成
相手への配慮が必要なメールや、言いにくいことを伝える際の文面作成を支援します。

### 📩 催促・依頼（角を立てずに伝える）
相手が期限を守らない、返信がないなどの状況で、関係性を壊さずに催促する場合のプロンプトです。

```text
あなたは気配りのプロフェッショナルである優秀な秘書です。
以下の {制約条件} と {状況} に基づき、取引先への「催促メール」のドラフトを作成してください。

# 制約条件
- 相手の状況を気遣う「クッション言葉」を使用すること。
- こちらが困っている事実は伝えつつ、相手を責める口調にならないこと。
- 次の具体的なアクション（いつまでに何をしてほしいか）を明確にすること。

# 状況
- 相手：プロジェクトの協力会社（長い付き合い）
- 課題：先週提出予定の見積もりがまだ届いていない
- 困っていること：明日、社内の決裁会議があり、それに見積もりが必要
```

### 🙇 謝罪・トラブル対応
ミスをしてしまった際、誠意が伝わる謝罪文を素早く作成します。

```text
あなたは危機管理広報の経験があるベテランの営業マネージャーです。
以下の {制約条件} と {トラブル内容} について、顧客へ送る「お詫びメール」を作成してください。

# 制約条件
- 言い訳をせず、事象と原因、今後の対策を明確にすること。
- 相手の怒りや不安に寄り添う、誠実なトーンであること。
- 箇条書きを使って、対策内容を分かりやすく伝えること。

# トラブル内容
- 納品したシステムに不具合があり、半日ほど利用できない時間が発生した。
- 原因はサーバーの設定ミス。現在は復旧済み。
```

## 2. 文章校正・リライト
書いた文章の誤字脱字チェックや、トーン＆マナー（語調）の変更を行います。

### ✍️ プロの校正（誤字脱字・修正）
単純なチェックだけでなく、修正箇所を一覧で出すことで確認作業を楽にします。

```text
あなたは出版業界で長年活躍するプロの校正者です。
以下の {制約条件} と {原稿} をチェックし、誤字・脱字、表記ゆれ、分かりにくい表現を修正してください。

# 制約条件
- 原文のニュアンスを極力残すこと。
- 修正した文章を提示した後、修正箇所と理由を表形式で列挙すること。

# 原稿
新弊社の新製品、ディジタルカメラの「ルミナスX」は、今月末より発売を開始致します。このカメラの最大の特長は、高感度センサーを搭載している点です。これにより、光量が不足している環境下でも、ノイズを極力抑えたクリアな写真を撮影する事が可能です。また、バッテリーの持続時間も従来のモデルと比べ、大幅に延長され、ストレス無く長時間の使用にたえる事ができます。特に、ユーザーインターフェース（操作画面）は、直感的な操作を念頭に設計されており、誰でもカンタンにプロ並みの写真が撮れるようになっています。ご期待下さい。尚、発売記念キャンペーンに付きましては、追ってご連絡させて戴きます。
```

### 🎨 トーン＆マナー変換（要約・言い換え）
硬い文章を柔らかくしたり、その逆を行ったりします。

```text
あなたは読み手に寄り添うWebライターです。
以下の {硬い文章} を、新入社員でも直感的に理解できる、親しみやすい文章に書き換えてください。

# 制約条件
- 専門用語には（）で簡単な解説を加えること。
- 「です・ます」調で、語りかけるような口調にすること。
- 重要なポイントは箇条書きにすること。

# 硬い文章
従業員は、会社の情報資産を保護する責務を負う。機密情報の取り扱いにおいては、アクセス権限の有無を確認し、定められた手順に従い、慎重な管理を徹底するものとする。特に、個人情報を含むデータについては、目的外利用の禁止を厳守し、情報漏洩のリスクを最小化するための是正措置を速やかに実行しなければならない。違反が確認された場合、就業規則に基づき懲戒処分の対象となる可能性がある。
```

## 3. 会議・ドキュメンテーション
会議の準備から事後の議事録作成までをサポートします。

### 📝 議事録の要約とネクストアクション
文字起こしテキストから、必要な情報だけを抽出します。

```text
あなたは論理的思考力に優れたプロジェクトマネージャーです。
以下の {会議ログ} を読み、チームメンバーに共有するための「議事録サマリー」を作成してください。

# 制約条件
- 以下のフォーマットで出力すること。
  1. 決定事項（何が決まったか）
  2. 保留事項（何が決まらなかったか）
  3. ネクストアクション（誰が、いつまでに、何をするか）
- 余計な挨拶や雑談はすべてカットすること。

# 会議ログ
Aさん: えーっと、じゃあ、新機能の「ダッシュボードV2」のUIデザインについて今日の会議を始めたいと思います。まず、全体的なコンセプトですが、Bさん、進捗はいかがですか？
Bさん: はい。ワイヤーフレームはほぼ完成しています。ただ、色使いについてちょっと迷っていて、現行の青基調を維持すべきか、それとも、最近のトレンドに合わせてダークモードを導入すべきかで意見が分かれてるんですよね。デザイナーのCさんはダークモード推しみたいで。
Cさん: 私はダークモードの方がユーザーエンゲージメントが高まると思うんです。特に夜間の使用を考えると目に優しいですし。でも、実装コストはちょっと上がるかもしれません。
Aさん: なるほどね。コストは重要だよね。えーと、Dさん、実装にかかる工数ってどれくらい変わりそう？ ダークモードの仕様って、まだ詰めてないから概算でいいんだけど。
Dさん: （少し考えて）そうですね。もしCSSの変数をうまく使えれば、そこまで大きな差は出ないかもしれません。ただ、既存のレガシーコード（古いプログラム）の部分に影響がないか、もう少し詳しく調査が必要です。とりあえず、来週の火曜までには正確な工数見積もりを出します。
Bさん: ありがとうございます。じゃあ、デザインの方向性は、来週のDさんの見積もりを見てから最終決定でいいですか？
Aさん: うん、そうしよう。**デザインカラーは一旦保留。**それと、ダッシュボードに表示するKPI（重要業績評価指標）の選定ですが、これはもう決まりましたよね？
Bさん: はい、ユーザーからのフィードバックを基に、「アクティブユーザー数」と「コンバージョン率」の2つをメインに表示することでチーム内で合意済みです。
Aさん: 了解。じゃあ、それは決定事項として進めてください。あと、Eさんから、マニュアル作成の件で質問があったと思うけど？
Eさん: あ、はい。マニュアルの作成を外注する件ですが、候補となるベンダー（業者）は3社に絞りました。それぞれの提案書をAさんに今日中にメールで送りますので、ご確認いただけますでしょうか。来週中には発注先を決めたいのですが。
Aさん: ありがとう。提案書、今日中にチェックして、明日にはフィードバックします。じゃあ、**Eさんは今週の金曜までに、各ベンダーに最終的な価格交渉の連絡を入れておいてください。**特に、納期の前倒しが可能かどうかに焦点を当てて。
全員: はい、承知しました。
```

### 📅 会議アジェンダ作成
効率的な会議を行うための進行表を作成します。

```text
あなたはファシリテーションのプロです。
以下の {会議の目的} と {前提} に基づき、1時間の会議アジェンダ（進行表）を作成してください。

# 制約条件
- 各パートの時間配分（分）を記載すること。
- 議論が発散しないよう、各パートの「ゴール（何が決まればOKか）」を併記すること。
- 最初のアイスブレイクと、最後のまとめの時間を含めること。

# 会議の目的
- 新しい経理システムの導入について、反対派の意見を聞きつつ運用ルールを決めたい。
- 参加者：経理部（導入賛成）、営業部（入力が面倒で反対）、システム部

# 前提
導入が提案されている新経理システムは「クラウド型経費精算システム」である。従来のシステムと比較し、以下のメリット・デメリットがある。
メリット:
　経理側の入力・確認作業時間が約 30% 削減される見込み。
　スマートフォンからの申請が可能。
デメリット:
　営業側は、外出先でのレシートの写真撮影とカテゴリ入力が必須となる。
　システム導入の初期費用が約 500 万円かかる。
参加者の背景：
　経理部： 業務効率化を強く望んでおり、導入に賛成。
　営業部： 外出先での入力作業増加を懸念しており、運用ルールの緩和を求め反対。
　システム部： 導入費用やセキュリティ面で問題はないが、運用後の問い合わせ増加を懸念。
```

## 4. 日常の壁打ち（思考整理）
アイデア出しや、考えを整理したい時に使います。

### 🧠 クイック・ブレインストーミング
質より量でアイデアを出したい時に。

```text
あなたは発想力豊かなプランナーです。
以下の {テーマ} について、アイデアを10個出してください。
実現可能性は一旦無視して、ユニークで斬新な視点を重視してください。

# テーマ
来月の社内イベント（懇親会）で、オンライン参加の人も楽しめる企画

# 出力形式
- アイデア名
- 概要（1行）
```
