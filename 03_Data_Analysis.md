# 03. データ分析・可視化（Data Analysis & Visualization）

このファイルでは、数値データやテキストデータをAIに読み込ませ、分析・可視化・インサイト抽出を行うためのプロンプトを集めています。

※ ChatGPTやGeminiのファイル読み込み機能で、ファイルをアップロードして使用することを想定しています。

---

## 1. 定量データの可視化（数値分析）
ExcelやCSVデータを元に、グラフ作成や傾向分析を行います。

### 📊 グラフ描画と課題抽出
売上データや人口データなどから、視覚的なグラフを作成し、そこから読み取れる課題を挙げさせます。

```text
# 命令書
あなたはデータ分析のスペシャリストです。
アップロードした {データファイル} を読み込み、以下の分析を行ってください。

# 手順
1. データの概要（カラム構成、欠損値の有無）を確認する。
2. 「渋谷区」の2013年からの「総世帯数」の推移を棒グラフで描画する。
   - グラフのタイトルや軸ラベルには日本語フォントを適用すること。
3. 作成したグラフから読み取れる「傾向」と、考えられる「今後の課題」を3点挙げる。

# 入力データ
（ここにファイルをアップロード）
```

### 📉 要因分析と施策提案
結果（数字）だけでなく、その背景にある原因を推察させます。

```text
# 命令書
あなたは事業戦略に強いデータサイエンティストです。
このデータは、ある動画配信サービスのユーザー行動ログです。
解約率が高いユーザー層を特定し、その原因を仮説ベースで分析してください。

# 制約条件
- 年代別、視聴ジャンル別などでクロス集計を行うこと。
- 分析結果に基づき、解約を食い止めるための具体的な改善施策を3つ提案すること。
```

## 2. 定性データの分析（テキスト分析）
アンケートの自由記述や、口コミなどのテキストデータを分析します。

### 🗣️ 顧客の声（VoC）分析
大量のテキストデータから、ユーザーの感情やニーズを抽出します。

```text
# 命令書
あなたは定性調査のプロであるマーケターです。
アップロードした {アンケートデータ} の「自由記述欄」を分析し、ユーザーが抱えている「不満」と「要望」を洗い出してください。

# 分析手法
- 頻出単語を確認するだけでなく、文脈を考慮して内容を分類すること。
- 似たような意見をグループ化し、件数が多い順にランキング形式で出力すること。

# 出力形式
| 順位 | 不満・要望のカテゴリ | 具体的な声（要約） | 件数（推定） |
| :--- | :--- | :--- | :--- |
```

### 📝 キーワード分類（KJ法）
バラバラのキーワードやアイデアを、意味の近いもの同士でグルーピングします。

```text
# 命令書
あなたは情報を整理・構造化するのが得意な編集者です。
以下の {キーワードリスト} は、SEOツールで抽出した「検索キーワード」です。
これらをテーマごとに分類（KJ法）し、それぞれのグループに適切な「ラベル（見出し）」を付けてください。

# ステップ
1. キーワードの意味を理解し、親和性の高いものをグルーピングする。
2. 各グループに、その内容を表すキャッチーな見出しを付ける。
3. 各グループを元に、記事コンテンツの企画案を提案する。

# キーワードリスト
（ここにCSVやテキストデータを貼り付ける）
```

## 3. 資料化（アウトプット）
分析結果を、そのまま報告書やスライドに使える形にします。

### 📑 分析レポート作成
グラフと考察を組み合わせた、完成度の高いレポートを作成させます。

```text
# 命令書
あなたは外資系コンサルタントです。
ここまでの分析結果（グラフ、課題、施策）をまとめ、経営層に報告するための「分析レポート」の構成案を作成してください。

# 制約条件
- 結論ファースト（Conclusion First）で書くこと。
- 専門用語は使わず、ビジネス視点での「示唆（Implication）」を含めること。
- この分析結果をスライド1枚にまとめる場合の「スライド構成（タイトル・ボディ）」も併記すること。
```
